# 抖音短视频爆款文案

抖音口播文案创作助手。两大路径闭环：爆款视频链接 → 提取文案+数据 → 分析爆点 → 生成新文案；话题/书籍 → 搜索素材 → 生成爆款文案。

## MCP 服务配置

本技能依赖以下 MCP 服务，运行时**必须**先读取 `mcp-config.json` 获取服务 URL：

| 环境变量名 | MCP 服务 | mcpId | 用途 |
|-----------|----------|-------|------|
| `$MCP_DOUYIN_VIDEO_URL` | 获取抖音视频数据 | 1076 | 从抖音链接提取视频文案、互动数据、博主信息 |
| `$MCP_BOCHA_URL` | 博查搜索 | 1039 | 搜索话题热点、爆款趋势、书籍核心观点 |
| `$MCP_METASO_URL` | 秘塔AI-链接速读 | 9400 | 读取用户提供的参考链接（书评、文章等）提取正文 |

### 配置读取方式

**约束：调用方 agent 必须在执行任何 MCP 调用前，先读取 mcp-config.json 文件**

```python
import json

with open("mcp-config.json", "r", encoding="utf-8") as f:
    config = json.load(f)

douyin_video_url = config["MCP_DOUYIN_VIDEO_URL"]["url"]
bocha_url = config["MCP_BOCHA_URL"]["url"]
metaso_url = config["MCP_METASO_URL"]["url"]
```

## 工作流程

根据用户输入自动判断走哪条路径：

- 用户给了**抖音视频链接/口令** → 路径A（爆款仿写）
- 用户给了**话题/书籍名称** → 路径B（原创生成）
- 用户**同时给了链接和话题** → 路径C（融合创作）

---

### 路径A：爆款视频仿写（链接 → 提取 → 分析 → 生成）

#### A1. 提取原始视频数据

```bash
# 读取配置
# 先发现可用工具
python3 scripts/call_mcp.py list "$MCP_DOUYIN_VIDEO_URL"

# 1. 从链接获取视频ID
python3 scripts/call_mcp.py call "$MCP_DOUYIN_VIDEO_URL" get_video_id \
  --params '{"vedio_url": "<用户提供的完整链接或口令>"}'

# 2. 用视频ID获取完整数据
python3 scripts/call_mcp.py call "$MCP_DOUYIN_VIDEO_URL" get_video_info_by_id \
  --params '{"vedio_id": "<上一步返回的视频ID>"}'
```

返回数据中重点提取：
- **视频文案**（原始口播文本）
- **互动数据**：点赞数、评论数、分享数、收藏数、播放量
- **博主信息**：昵称、粉丝量级

#### A2. 爆点分析

基于提取的原始文案，分析以下爆款要素：

1. **开头钩子类型**：悬念型 / 反常识型 / 痛点型 / 数字型 / 对比型
2. **情绪节奏**：文案的情绪起伏曲线（开头激发→中间共鸣→结尾升华）
3. **钩子密度**：每隔多少字出现一个"留人钩子"（理想值：每50-80字一个）
4. **口语化程度**：是否适合口播朗读，有无书面语需要转化
5. **互动引导**：是否有评论区引导（"你觉得呢""评论区告诉我"）

#### A3. 生成新爆款文案

基于分析结果，生成**同主题但不同角度**的新文案，要求：
- 保留原视频的爆点结构
- 更换切入角度，避免雷同
- 强化钩子密度
- 控制在600字左右

---

### 路径B：话题/书籍原创文案（输入 → 搜索 → 生成）

#### B1. 素材搜集

```bash
# 搜索话题相关热门内容和爆款角度
python3 scripts/call_mcp.py call "$MCP_BOCHA_URL" web_search \
  --params '{"query": "抖音爆款 <话题关键词> 口播文案"}'

# 如果是书籍，搜索核心观点和金句
python3 scripts/call_mcp.py call "$MCP_BOCHA_URL" web_search \
  --params '{"query": "<书名> 核心观点 金句 读书笔记"}'
```

#### B2. 参考链接精读（可选）

如果用户提供了参考链接（文章、书评、帖子等）：

```bash
python3 scripts/call_mcp.py call "$MCP_METASO_URL" reader_html_content \
  --params '{"url": "<用户提供的链接>"}'
```

#### B3. 生成爆款文案

基于搜集的素材，直接生成文案。

---

### 路径C：融合创作（链接+话题）

先走路径A提取和分析爆款视频，再走路径B搜索话题素材，最后融合两者生成。

## 爆款文案写作规范

### 结构模板（600字口播稿）

```
【黄金3秒钩子】（必须在前3秒抓住注意力，1-2句话）
↓
【痛点/共鸣引入】（让观众觉得"说的就是我"，2-3句话）
↓
【核心干货段】（3-5个要点，每个要点一句话，密集输出价值）
↓
【转折/反转/升华】（制造情绪高潮）
↓
【金句收尾+互动引导】（留下记忆点，引导评论/关注）
```

### 钩子类型库

| 钩子类型 | 示例 | 适用场景 |
|----------|------|----------|
| 悬念型 | "你知道为什么90%的人都做错了吗？" | 知识类/认知颠覆 |
| 反常识型 | "读了100本书我才发现，读书最没用的方式就是从头读到尾" | 读书/学习类 |
| 痛点型 | "月薪5000的你，还在用时间换钱？" | 职场/成长类 |
| 数字型 | "3个方法，让我一个月涨粉10万" | 方法论/干货类 |
| 对比型 | "普通人和高手之间，只差这一个认知" | 认知/思维类 |
| 故事型 | "昨天一个粉丝私信我，看完我沉默了" | 情感/故事类 |
| 恐吓型 | "如果你还不知道这件事，2025年会很难过" | 趋势/预警类 |

### 写作铁律

1. **每50-80字必须有一个钩子**：反问、转折、悬念、金句，不让观众划走
2. **禁止长句**：每句话不超过20字，口播节奏感
3. **口语化**：用"你""咱""说白了""讲真"，禁止书面语
4. **情绪词密集**：震惊、扎心、太真实了、细思极恐、建议收藏
5. **具体数字代替形容词**："很多人"→"90%的人"，"赚了不少"→"赚了50万"
6. **开头3秒定生死**：第一句话必须是钩子，不要铺垫
7. **结尾必须有行动指令**："关注我""评论区告诉我""转发给你的朋友"

## 注意事项

- 用户给抖音链接时，完整保留原始链接/口令，不要处理特殊字符，直接传给 MCP
- 视频文案提取后需要人工确认是否为口播文案（有些视频是纯画面无文案）
- 书籍类文案需标注"灵感来源：《书名》"，避免版权争议
- 如果某个 MCP 调用失败，不中断流程，用已有素材继续生成
- 生成的文案默认600字左右，用户可以要求调整字数
- 严禁抄袭原文案，仿写是"学结构、换角度、出新意"
